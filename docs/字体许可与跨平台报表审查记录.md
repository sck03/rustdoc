# 字体许可与跨平台报表审查记录

> 更新日期：2026-07-23
>
> 范围：正式 PDF、无头浏览器、浏览器打印预览、默认出口单据模板和桌面发布包。

## 1. 字体许可口径

正式报表只允许使用 `Resources/Fonts/OpenSource/font-manifest.json` 中登记的三个 Noto CJK 文件：

| 文件 | 用途 | 许可证 |
| --- | --- | --- |
| `NotoSansCJKsc-Regular.otf` | 正文、表格和无衬线标题 | SIL Open Font License 1.1 |
| `NotoSansCJKsc-Bold.otf` | 表头、重点金额和标题 | SIL Open Font License 1.1 |
| `NotoSerifCJKsc-Regular.otf` | 中文付款单正文 | SIL Open Font License 1.1 |

字体版本固定到 Noto CJK 上游 commit，并在 manifest 中记录 SHA-256。`.otf` 文件不提交 Git，由构建阶段下载、校验并放入程序根 `Resources/Fonts/OpenSource`；许可证、manifest 和 README 随包保留。

微软雅黑、Segoe UI、宋体、Arial、SF Pro、PingFang、Consolas 等只允许作为操作系统已有字体的 CSS 回退名称，禁止复制字体二进制进安装包。`verify-font-license-policy.mjs`、发布包 payload 审计和 CI 均会拦截未登记或哈希不匹配的字体。

## 2. 报表输出策略

- `ReportFontPolicy` 为无头 Chromium/PDF 导出注入本地 `@font-face` 和统一字体变量；字体路径来自程序资源根，不读取系统字体目录。
- 发票、箱单、合同、报关单统一使用 Noto Sans CJK；中文付款单正文使用 Noto Serif CJK，标题和表头使用 Noto Sans CJK，消除对 Arial/宋体自动回退的依赖。
- 浏览器 HTML 预览和打印预览继续复用同一份已渲染模板，等待 `document.fonts.ready` 与图片加载后再打印。浏览器直接打印属于快速打印入口；需要跨平台分页、换行和归档一致性的正式输出应使用“生成 PDF”入口，该入口走上述本地字体注入的 Chromium 链路。
- 技术文本统一使用前端 `--edm-font-mono` 等宽变量；唛头 Canvas/图片和报表模板分别保留明确的跨平台回退栈，不复制系统字体。

## 3. 版式基准与箱单规则

- 默认箱单保持每 12 条商品为逻辑页，长品名允许在“货品说明 / DESCRIPTION OF GOODS”列内自然换行，不再把品名锁死在固定单行。
- 品名独立一行，款号和订单号在其下各占一行；模板不擅自添加 `Style:` 或 `Order No.:` 前缀，用户在商品明细中录入的原文会原样显示。
- 混装行的箱数为零或空时不输出 `0CTN`，留空表示承接上一箱；最后一页显示按单位汇总的箱数、数量、重量和体积。
- 当前列名采用行业更易理解的双语名称：唛头、货品说明、包装、数量、毛重、净重、体积。列名不是数据契约，后续自定义模板可以按企业习惯改名。

## 5. 发布前门禁

1. 执行 `node scripts/provision-report-fonts.mjs`。
2. 执行 `node scripts/verify-font-license-policy.mjs --require-files`。
3. 执行 Web 构建、无障碍、视觉和缩放契约。
4. 执行 .NET 报表 PDF 回归并确认没有 `U+FFFD`、空字符、截断或跨页重叠。
5. 用 `scripts/verify-package-payload.ps1` 检查发布包只含 manifest 登记字体，并确认浏览器开发 UI 字体已从正式包移除。

许可证全文见 `Resources/Fonts/OpenSource/OFL-Noto-CJK.txt`；关于页同步展示字体家族和许可证信息。
