# 单证制作软件设计规划
> 最后更新: 2026-06-29
>
> 本文只保留当前有效的长期设计基线；阶段性改造记录见 `程序改进文档.md`。

## 1. 项目定位

本项目是面向外贸 SOHO、单证员和小型外贸团队的本地优先桌面单证系统。核心目标是把发票、商品、付款、报表、邮件、信用证审查、装柜估算和单一窗口对接组织成一套可验证、可维护、可离线工作的业务流。

当前技术基线：

- `.NET 8 / C# 12 / ASP.NET Core API sidecar / Tauri v2 / React / TypeScript / WebView2`
- `Domain + Application + Infrastructure + API + Tauri/Web`
- `EF Core + SQLite(SQLCipher) / PostgreSQL`
- `HTML + Scriban` 模板
- `Serilog / PaddleOCR V6 Small ONNX CPU x64 离线模型目录 / Mapperly`

项目尚未部署生产环境，因此新改造默认直接替换旧实现，删除无调用方胶水，不为未投产旧代码保留兼容壳。

## 2. 架构边界

`Tauri/Web` 层只负责桌面宿主、路由导航、页面布局、交互状态、文件/目录选择、只读详情和窗口关系。桌面能力统一通过 Tauri command 暴露给 React 页面，业务能力统一经 API/Application 服务进入；页面不得直接写数据库、直接拼运行目录或绕过 API 调用 Infrastructure。

`Application / API` 层负责页面状态所需的流程编排与业务契约：Busy 状态、按钮可用性、分页搜索、选中项恢复、保存删除、导入导出、预览发送、脏数据跟踪和关闭前确认均应有明确接口或 DTO。已注册服务必须通过构造函数显式注入，缺依赖应在装配期暴露。

`Service` 层负责业务能力：持久化、编号、冲突处理、模板生成、PDF/打印、邮件、更新、OCR、汇率、装柜、审计、单一窗口字段映射、XML、交接包、回执解析和状态回写。

`Infrastructure / Utils` 层负责共享基础设施：运行路径、数据库、读仓、安全落盘、原子 ZIP、分页导出、请求协调、本机偏好、HTML 文件导航、临时文件清理、OCR/浏览器/模板资源、授权验签和平台本机安全能力。

数据库事务入口统一走 `AppDbContextExecution.ExecuteInTransactionAsync(...)`。该 helper 会使用 EF Core provider 的 execution strategy 包裹“创建上下文、开启事务、执行业务、提交/回滚”全过程，避免 PostgreSQL 启用连接重试后与手写事务冲突。业务服务不得再直接调用 `BeginTransactionAsync()`。

默认配置路径、默认本机偏好路径和默认数据库连接配置都通过显式无参入口表达；传入自定义路径或连接配置时必须传真实对象，不使用 `null` 作为“走默认配置”的哨兵值。

Excel 导入模板属于正式资源，不作为线下临时附件管理。模板文件随程序发布包进入程序根 `Resources/ExcelTemplates`，前端 Excel 工具只承接数据导入、导出空白完整导入模板和导出空白托单模板；已填写 Excel 派生为托单副本属于文件转换动作，放在“模板与工具 -> Excel 工具”。用户使用的是程序派生出的副本，不直接暴露内置模板目录，避免模板版本与程序导入规则脱节，也避免误改内置资源。

订舱托单是导入数据模板的展示派生物，不另起一套字段事实源。转换时保留原始数据和工作簿格式，只隐藏货代订舱通常不需要的内部导入列（HS 编码、境内货源地、英文单位、中文单位、箱数单位），并扩宽右侧标签列，保证隐藏列后船期、贸易条款、目的港等信息仍可读。转换结果必须另存为新文件，不能覆盖源 Excel。发票列表中的“导出货代订舱托单”也走同一模板服务：先按当前 Excel 导入模板字段口径把选中发票和商品明细写入内置模板副本，再应用订舱托单布局。

系统设置中的“导入模板默认出口商中文名称”属于该模板副本的业务默认值。导出模板时优先使用该配置；未配置时，仅当数据库中存在唯一一个出口商中文名称才自动带出；否则回退为“请填写出口商中文名称”占位文本，避免空库或多出口商场景误导用户。

## 4. 模块边界

### Main Shell

主工作区由 Tauri 桌面壳与 React 页面共同承接：Tauri 负责 sidecar 生命周期、运行路径、桌面文件能力和安装交接；React 负责菜单入口、路由、页面装配、更新检查和工作区状态。登录后固定进入仪表盘，不持久化最近停留页。主菜单按“工作台 / 单证业务 / 单一窗口 / 基础资料 / 模板与工具 / 系统维护”分组，分组下再展开子菜单，避免所有功能平铺成同级入口。

### Invoice / Item / Payment

发票、商品和付款继续按 `React 页面 + API DTO + Application Service + ReadRepository` 维护。写服务只保留保存、删除、复制、编号、冲突处理和单据级读取；列表、搜索、分页和导出固定走读仓。发票删除必须加载带 `RowVersion` 的实体，并在同一事务清理商品明细和单一窗口派生工作区。

发票、商品明细、付款和查询页面继续保持前端组件、API DTO 与 Application 服务边界清晰。新增动作进入对应页面模块、API handler、Application use case 或 helper，不恢复无参构造、可省略服务参数、空服务 fallback、行级自动修复或整批读取路径。

信用证审查的文本导入由 `ILetterOfCreditDocumentService` 统一处理：文本/PDF 可直接抽取时优先使用结构化文本，图片和扫描 PDF fallback 固定经共享 `IOcrService`，不得在信用证链路单独引用 PaddleOCR 模型或保留旧 OCR 分支。

### Master Data

客户、出口商、收款对象、商品、端口、单位、HS 编码和辅助资料的列表/搜索固定走 `LocalMasterDataReadRepository` 对应接口。写服务只承接写入、删除、单条读取和业务规整。主数据页面按分类、列表、详情编辑、搜索和危险操作分区组织，避免把所有主数据维护挤在同一表单；右键/批量/删除/保存外围交互优先复用前端通用组件、API 契约和 Application helper，不在每个页面复制同一套胶水。HS 编码远程抓取、导入、本地读取和推荐链路按职责文件维护。

### Reporting

报表模块由模板目录、模板包、HTML 渲染、预览、PDF、打印和邮件组成。模板目录由 `ReportTemplateCatalog` 承接，报表实体装配由 `ReportDataAssembler` 承接，HTML 预处理和 Scriban 渲染由 `ReportTemplateRenderer / ReportTemplateGlobalBuilder` 承接，PDF 合并后处理由 `PdfMergeService` 承接。`ReportService / ReportDataAssembler` 必须显式依赖真实 `IDbContextFactory<AppDbContext>`，不接受空工厂作为模板目录操作的旁路。打印预览由 Web 预览、浏览器打印和 Tauri/API PDF 生成闭环承接，通知和文件对话框能力缺失必须在装配期或页面能力检测时暴露。

查询、审计等高频列表导出必须分页读取、分段写表、固定列宽，不使用全表自动列宽扫描。大 HTML 预览和 PDF 生成走共享临时 HTML 文件导航，不直接依赖 `NavigateToString(...)`。

### Tools

装柜、OCR、AI、汇率、审计日志和更新链路属于工具能力。装柜算法在服务层，React 页面只编排状态、项目读写、柜型维护和分析刷新；装柜持久化、通知、提示和算法引擎均为构造期必填服务。更新中心固定通过设置服务读取更新源，未配置更新源属于业务状态，缺少设置服务属于装配错误。OCR 固定从程序根 `OcrModels/PaddleOCR/V6` 加载随包发布的 PP-OCRv6 Small ONNX 检测、识别和 `inference.yml` 字典文件，初始化时必须校验模型类别数与字典对齐；OCR/AI 继续优先补取消、超时、错误可观测性和资源释放边界。

### System Settings

业务设置和本机偏好分文件维护。设置写盘走串行更新或防抖写回；本机偏好按文件路径合并落盘。列布局、分栏、最近目录和筛选状态必须显式依赖 `ILocalPreferenceService`，不保留缺服务静默跳过路径。

设置中心按成熟商用软件的信息架构拆成“运行与数据库 / 单据与模板 / 邮件与备份 / AI 与单一窗口 / 用户与权限 / 维护工具”分类。全局保存、刷新、恢复默认、校验设置、日志清理和敏感字段开关集中在顶部操作条，右侧只显示当前分类内容；新增设置动作不得回流成一个超长页面。

用户与权限管理归入基础数据/系统设置边界：页面只做管理员权限检查、列表选择、账号字段编辑和删除确认，保存/删除规则由 API/Application 与 `IUserService` 承接。账号编辑区保持紧凑、可扫描，长标签不得挤压输入区。删除用户只用于清理误建且无业务归属的账号；当前登录账号、最后一个启用管理员以及已有发票/付款归属的账号不得删除，应改用停用账号保留历史归属。

单一窗口词典维护按“页面注册 / Excel 行导入 / 页面适配器与行模型 / 表格交互 / 持久化”分段。新增码表页时只扩展对应适配器和导入器，不把行模型、Excel 读取和去重逻辑重新塞回页面注册文件。

PostgreSQL 共享库配置优先按 `服务器 / 端口 / 数据库名 / 账号 / 密码 / 附加参数` 分字段维护，由应用统一生成连接串；普通局域网场景不额外暴露“网络配置”字段，只要求当前电脑能访问目标服务器和端口。数据库密码与邮件/WebDAV 密码一样按本地加密方式保存。

切换 PostgreSQL 后，所有电脑应指向同一台服务器和同一个业务数据库；服务器、端口、数据库名通常一致，账号密码可按岗位统一或分账号配置。每台电脑仍保留自己的本机设置、最近目录、窗口偏好和单一窗口客户端目录，因为这些是机器级状态，不应写成全公司共用。

当前 PostgreSQL 共享库已经具备连接、装配、事务执行策略、单一窗口协同基础和第一层业务归属能力。发票与付款已带 `OwnerUserId / DepartmentId / CompanyScope`，共享库模式下普通账号的发票、付款、查询页、单一窗口操作中心、协同工单和跟踪写入默认只处理本人归属数据，`Admin` 可看全部；基础数据菜单已提供管理员可用的用户与权限管理入口，可维护用户名、姓名、角色、部门、启用状态、初始/重置密码，并删除无业务归属的误建账号。PostgreSQL 新库首次初始化时必须在登录窗口为 `admin` 输入初始密码，空密码不会创建或登录共享库管理员；SQLite 单机库仍允许默认空密码。`CompanyScope` 当前只作为归属字段保留，不在用户管理界面暴露为可配置权限。下一步仍需补部门/公司范围授权策略、归属改派/批量补属工具、PostgreSQL 备份/还原和更完整的公司级管理报表。没有这些管理能力前，PostgreSQL 可以作为受控团队共享库试运行，但不应直接当成完整公司级权限后台。

SQLite 单机版继续作为默认模式没有结构性问题。归属字段会随模型进入 SQLite，但查询、保存、删除和单一窗口跟踪不启用共享库归属过滤，单机库保持全量可见。项目尚未部署生产环境，数据库初始化按当前模型创建新结构；不再为未投产旧库保留专门升级胶水，结构变化时优先重建本机库或按当前导入/导出能力迁移数据。

### SingleWindow

单一窗口是受控扩展模块，不假设每台电脑都能正式提交。当前支持 `COO / ACD` 的本地闭环：编辑、字段映射、XML、交接包、回执导入、状态回写和操作中心。

单一窗口新增业务应通过“业务适配器 + 字段映射 + XML/文件生成 + 回执解析器”扩展，不把规则塞回发票主链路。文档持久化、交接包导出、导出审查、协同跟踪和客户端桥接必须显式依赖真实服务。协同数据源查询前固定登记当前工作站，不能把工作站注册服务作为可选能力绕开。

编辑器字段分组、提示文案、必填规则、清理规则、明细输入归一化、顶部摘要、审查导航、工具动作、字段锁定和源资料变化摘要优先集中到 catalog/helper/base class。前端页面只保留事件绑定、页面组合和控件生命周期。导出预检面板按主入口、布局和状态行为分段维护；新增审查列、筛选或动作时进入对应模块，不把表格列、按钮行为和结果映射重新堆回主入口文件。附件选择、生产企业资料选择和提交包/回执包路径选择统一走 Tauri desktop command 或浏览器安全文件输入，不在业务服务里直接调用系统对话框。

操作中心只维护业务目录根、发送、收件、回执打包和批次详情。路径解析、回执包默认命名和文件对话框偏好 key 走专用 helper，不创建旧式兼容目录。

协同页按状态入口、加载分页、视图偏好和行模型分段维护。后续新增协同字段、筛选项或状态列时进入对应模块，不把查询、偏好保存和 UI 行 DTO 重新堆回主入口文件。

## 5. 新代码约束

- 不在生产代码中保留 `?? new Xxx()`、可省略服务参数、空读仓、空服务 fallback 或服务级 `?.` 弱运行分支。
- API、Application 和桌面命令只保留真实装配入口；测试需要的替身由测试工厂显式传入，不让生产代码为测试保留短重载或 `null` 兼容路径。
- 默认路径/默认配置使用显式无参构造或无参重载表达；自定义路径、连接配置和服务依赖不接受 `null` 表达默认行为。
- DI 装配保持模块扩展、显式类型清单和共享注册 helper；同一实现暴露多个接口时使用 alias helper 表达关系，不把服务注册、窗体注册或测试替身写回启动入口。
- 文件/目录选择统一经 Tauri desktop command 或浏览器安全文件输入；业务专用弹窗统一经页面组件/桌面命令封装，不在业务服务里直连平台文件对话框。
- 不为未投产旧草稿、旧包格式或旧目录结构保留专项迁移分支。
- 数据库连接配置属于装配事实。首次缺少配置文件时使用显式 SQLite 默认值；配置文件损坏、缺少 `System` 节点、数据库类型拼写错误或 PostgreSQL 核心连接字段缺失时必须失败提示，不得静默切回 SQLite。
- 自定义数据库设置路径、自定义数据库连接配置和数据库模式判断不得用 `null` 表达默认值；默认配置只走显式无参入口。
- 需要跨多次 `SaveChanges`、先删后插、批量删除、状态回写或多表一致性的写操作必须使用 `AppDbContextExecution.ExecuteInTransactionAsync(...)`，不在业务服务内手写事务模板。
- 大体量 React 页面、API endpoint、Application service 或 Infrastructure service 按职责拆成组件、模块、helper、catalog 或 coordinator。
- 同类列表页复用共享分页、搜索、回选、旧请求取消和导出骨架。
- 共享分页搜索和列表查询能力必须显式依赖通知/错误处理与 API client，缺依赖在装配期暴露，不为列表页保留无参构造或空通知降级路径。
- 派生文件统一走安全落盘/原子替换；ZIP 创建和解压由 `ZipArchiveHelper` 承接。
- PDF 合并、ZIP 打包、模板包和交接包这类派生文件后处理必须透传取消令牌，并通过专用服务/helper 写入，不回流到批量导出主流程。
- 文件复制链路统一走 `FileCopyHelper`，有取消能力的链路必须透传 `CancellationToken`。
- 批量导出命名和同名避让只走 `BatchExportPathHelper`。
- 程序界面不持久化最后停留页，打开时固定仪表盘或当前业务深链明确指定的页面。
- 临时目录、运行时数据和调研抓取产物不得作为源码事实源。
- Windows 桌面版由 `scripts/run-tauri-local.ps1 build` 生成 Tauri 主程序，并通过 `scripts/prepare-tauri-bundle.mjs` 发布 API sidecar 与程序根资源；完整便携运行目录整理到 `artifacts/windows-desktop-run/ExportDocManager/`。离线授权注册机由 `apps/license-keygen-tauri` 生成，最终工具位于 `Tools/ExportDocLicenseKeyGen.exe`。旧 WinForms 发布脚本已下线。

## 6. 稳定性与性能基线

- 高频列表分页或按需加载，不做无上限整批读取。
- 高频列表导出分页读取、分段写表、固定列宽。
- 搜索和分页遵守“最新请求优先，旧请求取消或忽略”。
- 主工作区按路由懒加载页面，常用入口可做轻量预取；用户真实点击优先于后台预取，旧请求必须取消或忽略，避免隐藏页面继续和用户操作争抢 API、数据库或主线程资源。
- 页面切换遵守“轻切换、重数据治理”原则：重页面组件、基础资料读取、第一页查询、三维场景和大表格必须懒加载、幂等复用并可取消；页面容器应有稳定尺寸和可滚动区域，避免设置页、任务页、装箱 3D 舞台或明细表格把整体页面撑开。
- 后台预取必须可取消且资源释放必须幂等，窗口关闭、路由切换或页面卸载后不得继续更新已卸载组件。
- 页面切换前如需处理脏数据，必须先阻止跳转，再异步确认保存/放弃，确认完成后由代码进入目标路由；不得让浏览器默认导航吞掉保存确认。
- 商业桌面软件的顺滑切换通常来自组合方案：页面实例保活、数据分页/虚拟化、异步预取、主线程少做 IO、固定布局尺寸和轻量视觉反馈。本项目优先借鉴这些做法，避免用动画掩盖真实阻塞。
- 用户取消必须按取消退出，不包装成普通错误，不继续执行后处理。
- 页面卸载、退出登录或关闭窗口后，旧请求不得更新 UI。
- 审计日志、文本日志、备份、报表临时 HTML/PDF 和其他临时目录清理走共享 helper。
- 长生命周期联网服务复用 `HttpClient` 或等价客户端。
- OCR 模型、字典、图片、字体、WebView2/Chromium 和原生资源必须明确释放；PP-OCRv6 字典解析不得裁剪全角空格、半角浊点等合法字符，模型类别数不匹配时必须在初始化阶段失败。
- 浏览器/PDF 渲染运行时体积控制在发布阶段完成：保留当前架构、中文/英文语言包和基础运行链路；不需要的多语言包、反向架构桥、DRM 和 shell 预览组件按打包脚本瘦身。源码目录中的官方原始运行时不直接改写。
- 更新包下载、校验、解压和替换分阶段执行，替换必须等待主进程退出。
