# 报表模板多用户权限设计与实施记录

> 更新日期：2026-07-15
>
> 适用范围：Full Web / Docker 容器版，以及需要在 SQLite 单机版中预配置多个账号的场景。

## 1. 本轮结论

报表模板采用“默认文件模板 + 用户数据库模板”双层模型：

| 模板来源 | 存储 | 适用对象 | 修改权限 |
| --- | --- | --- | --- |
| 默认/管理员文件模板 | 程序根 `Templates/Export`、`Templates/Internal` | 全部有报表查看权限的账号 | 仅 `document.reports=manage` |
| 用户新建模板 | 当前运行数据库 `UserReportTemplates` | 创建者、明确共享的团队成员 | 创建者可编辑/删除；共享成员只读 |

这样既保留桌面版当前模板文件兼容，又避免容器多用户共同写一个程序目录文件造成越权和覆盖。

## 2. 用户操作规则

1. 普通用户进入“报表设计”后可以选择“默认模板”或“我的 / 共享模板”。
2. 点击“参考当前默认模板创建”时，服务端读取同一 `ReportType` 的默认模板内容作为初始草稿；用户也可以从当前共享模板“复制为我的模板”。
3. 新建模板默认私有。勾选“创建后共享给团队”后，其他账号能查看和复制，但不能直接保存、重命名或删除原模板。
4. 共享模板的使用者若需改版，必须复制为自己的模板；原作者的版本号和内容不变。
5. 创建、保存、删除、共享均在服务端执行所有权检查，前端按钮隐藏/禁用只是体验优化，不是安全边界。
6. 所有者和管理员可在设计器中“发布给团队 / 取消团队共享 / 停用模板 / 重新启用”；停用模板不会被真实预览和输出服务读取。
7. 共享范围直接使用当前模型的 `Private / Department / Company / All` 四值，不保留旧字段兼容或运行期猜测；项目尚未投产，数据库结构调整以当前模型一次性定型。

## 3. 报表类型与数据域隔离

`ReportType` 只有两个值：

- `ExportDocument`：报关单证，允许 `Invoice.*`、`Customer.*`、`Exporter.*`、`item.*` 等字段。
- `PaymentVoucher`：付款/报销，允许 `Payment.*`、金额换算和收款/经办字段。

共享范围说明：

- `Private`：仅模板所有者。
- `Department`：同公司、同部门的启用账号。
- `Company`：同公司启用账号。
- `All`：所有启用账号；共享模板仍然只读，需复制后修改。

服务端保存用户模板前执行交叉字段阻断：付款报销模板出现 `Invoice.`、`Customer.`、`Exporter.`、`item.` 或 `Invoice.Items` 会拒绝；报关单证模板出现 `Payment.` 会拒绝。前端新版设计器仍保留结构化 schema 校验，形成“前端即时提示 + 服务端最终阻断”两层保护。

## 5. 权限矩阵

| 操作 | 无 `document.reports=view` | `view` | `operate` | `manage` |
| --- | --- | --- | --- | --- |
| 查看默认模板 | 403 | 允许 | 允许 | 允许 |
| 查看自己的/共享模板 | 403 | 允许 | 允许 | 允许 |
| 新建自己的模板 | 403 | 403 | 允许 | 允许 |
| 编辑/删除自己的模板 | 403 | 403 | 允许 | 允许 |
| 共享自己的模板 | 403 | 403 | 允许 | 允许 |
| 修改他人共享模板 | 403 | 403 | 403 | 管理员可通过全量维护流程处理 |
| 修改程序根默认模板、模板包、目录探针 | 403 | 403 | 403 | 允许 |

内置“单证人员”模板的 `document.reports` 已从 `view` 调整为 `operate`，因此普通单证员可以设计自己的模板；Finance 保持 `manage`，Sales 不开放单证报表模块。

版本恢复不会覆盖旧版本，而是创建一个新的当前版本，例如从 V1 恢复后生成 V3；保存请求携带 `expectedVersion`，若其他窗口已经保存新版本，服务端返回 409，前端提示刷新后确认差异，避免静默覆盖。

现有 EF 审计拦截器会记录用户模板、版本、发布状态和删除动作。为符合商业软件安全要求，审计值执行统一脱敏：密码、密钥、令牌记录为 `[REDACTED]`；模板 HTML 和超过 2048 字符的长文本只记录长度与 SHA-256 摘要，不把完整内容复制进审计表。审计页面对用户模板实体以及新增/修改/删除动作使用中文名称展示。

## 6. 运行目录与数据库审查

- 用户模板内容写当前运行数据库；SQLite 默认文件仍位于运行目录 `App_Data/Database/data.db`，PostgreSQL 由管理员配置外部服务。
- 不新增 `C:\Users`、`AppData`、`ProgramData`、系统临时目录或浏览器 `localStorage` 持久化副本。
- 默认文件模板仍只读程序根 `Templates/`；从默认模板复制只读取内容，不修改默认文件。
- API 上传/保存内容受长度限制；路径型默认模板复制仍由既有 `ReportTemplatePathResolver` 约束在报表类型目录内。
