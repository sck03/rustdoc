## 1. 当前结论

新版报表设计器已完成阶段 1 的基础模块和实验入口，阶段 2 的文本/字段/轻量条件块/真实预览闭环，阶段 3 的明细表列编辑、列排序/复制、非循环侧栏、表尾合计行、分组小计行和分组另起页切片，阶段 4 的打印 CSS 基础补强、schema v2 页眉/页脚 band 打印模型、真实 Chromium 打印回归以及 A4/A5 横版/自定义纸张 PDF MediaBox 回归，并完成阶段 5 的保存/回读核心闭环与 schema migration/保存前校验切片。当前实现目标是把新版设计器从旧 Luckysheet/源码编辑器中拆出来，形成独立 React 模块、结构化 schema、基础画布、选择状态、撤销/重做、字段插入、拖拽式组件/字段放置、条件显示、真实预览、HTML/Scriban 导出、schema 注释回读和保存前结构保护能力。

报表模板页的管理入口已进一步收敛: 顶部和左侧不再常驻新建、重命名、删除、模板包导入导出等低频操作，这些动作折叠到“模板操作”和“模板包”面板中；当前模板的重命名输入也收进折叠项，避免和模板下拉重复常驻，顶部模板下拉宽度已缩短以减少对操作区的占用。模板包主流程按运行环境分流，桌面端保留“导出包/导入包/策略”，点击“导出包”会先打开 Tauri 保存对话框；浏览器端保留“下载包/上传包/策略”；按路径导出/导入保留在“高级路径”折叠区。真实数据预览也不再要求普通用户填写发票 ID 或付款 ID，而是按当前报表类型提供最近发票/付款报销单下拉，展示单据编号、客户/收款方、日期和金额，并保留业务页面跳转时携带的当前单据作为兜底。新版设计器右侧预览面板已拆出“样例数据”和“当前单据”两个数据模式: 样例数据默认继续调用既有 `previewReportTemplateContent` 后端样例渲染，同时新增样例档案下拉，可切换出口单据常规样例、长明细分页样例和付款票据样例；这些本地样例只在前端内存中把当前导出的 HTML/Scriban 做受控字段和明细循环替换，用于快速检查模板结构、长明细换行和固定票据格，不替代最终 API/Scriban/Chromium PDF 预览事实。当前单据模式只在结构化 HTML 已应用并保存同步后启用，复用既有发票/付款 HTML 预览端点核对已保存模板的真实业务数据效果。新版设计器出口单据默认草稿已改为 A4 竖版，付款/报销默认也保持 A4 竖版；主体默认按当前纸张高度组织，左右侧栏与纸张高度齐平并在内部滚动，避免侧栏比主体过长。

新版设计器当前可显式“应用到源码”或“保存模板”，保存仍复用既有模板保存 API，schema 嵌入模板 HTML 注释，不新增后端接口、数据库表或 sidecar 文件。旧 Luckysheet 可视化设计器、iframe 宿主、`/designer/*` 静态托管和 `Resources/Designer/` 离线资源已清理，报表模板页只保留新版设计器和源码模式；旧 HTML 模板不会自动转换，首次从新版设计器应用或保存结构化 HTML 前必须确认覆盖。当前 schema 已升级到 v2，读取 v0/v1/缺版本草稿会走 migration 和结构校验；缺失版区打印属性的旧结构默认保持非重复输出，也不会凭空补版区高度，避免静默改变旧模板打印行为。选中组件已支持上移、下移、复制、删除和键盘快捷键；所有 block 已新增 `output.enabled` 和 `output.note`，用于临时停用某个组件的 HTML/PDF 输出并保留设计备注。该能力借鉴成熟报表工具里的对象启用/备注属性，但不引入表达式引擎: 停用组件在画布和模型树中仍可见、可选中、可再次启用，保存导出时不生成实际报表 HTML。左侧侧栏已改为“组件库 / 报表模型 / 字段目录”三段互斥切换: 组件库提供文本、多列行、固定票据表格、条件块、图片/印章、明细表和分页符等受控报表组件，既可点击插入，也可拖入画布或模型树指定位置；报表模型树按 Header、Body、Footer band 展开 block，并支持直接选中、版区选中和原生拖拽跨版区重排；模型树现在会标注 Header/Body/Footer 的页眉/主体/页脚 band 角色，明细表会继续展开数据源、跨页表头、非循环侧栏、分组表头、分组小计、分组另起页和表尾合计等子节点，也会标记停用输出和设计备注；字段目录独立成段，不再被长模型树压到不可见，继续按业务分组插入字段，既可点击插入，也可拖到画布或模型树生成 Field block。右侧栏已从“属性 / 预览”拆为“页面 / 版区打印 / 组件属性 / 预览”四个明确入口，点击画布或模型树中的组件/版区会自动回到组件属性入口；画布也可选中版区本身，组件属性入口会切换为选中版区属性，可编辑版区最小高度、跨页重复、版区避免拆分、页脚短页贴底，并显示该版区组件摘要。字段目录已直接使用后端字段目录，不再用旧白名单过滤；出口单据显示 `Invoice/Customer/Exporter/item` 字段，付款/报销显示 `Payment.*` 和金额换算字段，普通用户可在字段输入框候选列表中选择字段名。`Row` 多列行组件已支持自由设置列数、列宽、列宽条拖动/方向键微调、单元格字段/固定文本、单元格样式和四边边框，适合发票同一行左右不同内容；`Grid` 固定票据表格用于截图这类复杂票据格，支持自定义列数/列宽、列宽等分、列宽条拖动/方向键微调、行高、增删行、增删单元格、横向/纵向合并、固定文本、字段、勾选组、竖排文字、样式和边框，并可把默认单元格样式或表格边框批量套用到全部单元格，因此可以自行搭出工厂付款单、费用报销单和审批单，而不是只能套用某个固定样式。明细表继续限定为出口单据 `Invoice.Items` 循环，付款/报销模板不能插入出口单据明细表，防止两个业务数据库字段混用；明细表作为主体明细 band 现在只能放入 Body，分页符也只能放入 Body，工具栏插入、拖拽插入、拖拽移动和保存前校验复用同一放置规则，防止把循环明细或分页控制放入重复页眉/页脚；明细表已支持非循环唛头侧栏、表尾合计行、分组表头、分组小计行、分组另起页、明细单元格组合表达、跨列表头、虚线/实线/无边框控制、列拖拽重排、列宽等分、列宽条拖动/方向键微调、列级边框批量套用和清除列级覆盖，其中组合表达由固定文本、明细字段和换行片段组成，分组小计由受控 `Sum`/`Count` 单元格生成 Scriban 变量，分组另起页只在第二个及之后的分组切换时输出受控打印 class，不向普通用户暴露任意表达式或 CSS。列宽条只调整相邻两列并保持总宽和最小宽度约束，仍写回结构化 schema 中的百分比或 `mm` 宽度。Header/Footer 版区已具备最小高度、跨页重复、版区避免拆分和页脚短页贴底属性，导出时通过 HTML/CSS 页壳和受控 `min-height: Nmm` 表达 page band，而不是引入商业报表格式。

本轮补充了更贴近成熟报表工具对象树的数据源视角: `reportDesignerModel.ts` 现在会汇总报表类型、纸张、业务数据源、字段绑定数量和停用组件数量；左侧报表模型面板顶部展示该概览，每个组件节点也会展开实际字段绑定。模板系统字段仍可在组件节点中看到，但不会被误算为业务数据源；这些信息只来自当前浏览器内存中的结构化 schema，不改变 HTML/Scriban 输出格式或保存路径。

前端构建已完成一轮 chunk 体积治理: 主应用路由改为懒加载，导航/登录壳与各业务页面拆分；Vite 增加 React/Router/Query、lucide 图标、Three.js 的手工 vendor 分包。当前剩余最大块为按需加载的 `vendor-three`，仅在进入装箱三维视图时拉取；Vite chunk warning 阈值调整为 600 kB，避免 3D 引擎块刚过默认 500 kB 时反复提示，同时保留对异常大包的提醒。

## 4. 技术选型口径

新版设计器不是旧 Luckysheet 设计器的二次包装，也不把 GrapesJS/Craft.js 作为最终依赖。旧 Luckysheet 可视化入口和离线资源已清理，后续主线是结构化 schema + React 画布 + HTML/Scriban 输出。

后续可以引入现代维护库，但必须符合以下条件:

- React 18/后续 React 版本兼容，TypeScript 类型完善。
- 免费开源、维护活跃、许可证清晰、无商业格式锁定。
- 能在 Tauri/Web 离线打包中稳定运行。
- 不接管本项目的报表 schema、分页模型或 HTML/Scriban 导出模型。
- 不把模板保存到新的数据库表、sidecar 文件或系统用户目录。

优先评估方向:

- `@dnd-kit/*`: 当前已用原生 HTML5 drag/drop 覆盖组件/字段拖入、block 跨版区重排和明细列拖拽排序，列宽可视化调整已用原生 pointer/键盘事件覆盖；后续若要做更复杂触控体验、跨容器嵌套拖拽或完整键盘可访问拖拽，再评估是否引入。
- Zustand 最新稳定版本: 仅在跨面板状态复杂后再考虑。
- Immer patches 或同类现代 patch 工具: schema 撤销/重做和可测试变更记录。
- Zod 最新稳定版本: schema 校验和导入/保存前提示。
- Tiptap/ProseMirror 生态: 仅在确需富文本块时引入，并限制输出范围。
