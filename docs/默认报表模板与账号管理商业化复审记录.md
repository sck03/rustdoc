# 默认报表模板与账号管理商业化复审记录

> 日期：2026-07-17  
> 范围：默认出口发票、默认装箱单、新建报表起始模板、Web/Tauri/容器账号管理、首次管理员初始化、路径与跨平台边界。

## 1. 本轮结论

- 默认发票和装箱单只移除固定英文前缀 `PO:`、`Style:`，订单号 `PoNumber` 和款式号 `StyleNo` 字段仍按原位置显示；字段为空时继续隐藏空内容。
- `Full` 管理员获得独立的“系统维护 → 账号与权限”导航和 `/system/access-control` 页面；系统设置不再重复嵌入用户管理分类。
- Web/容器版默认关闭公开自注册，采用“部署管理员首次使用 `admin` 登录初始化 → 管理员创建团队账号”的内部商业软件方案。
- PostgreSQL 空数据库继续由部署管理员第一次使用 `admin` 登录时设置首个管理员密码；其他用户名不能触发初始化。
- 本轮未引入新的 npm、NuGet、Cargo、Python 或外部认证服务依赖。
- 容器数据库默认升级为官方稳定 `postgres:18-bookworm`；数据卷按 PostgreSQL 18 新布局挂载到 `/var/lib/postgresql`，初始化使用内置 `PG_UNICODE_FAST` Unicode locale。Alpine 可用于受限测试但不作为商业默认值，官方没有 `postgres:18-slim` 标签。

## 2. 默认模板通用化

涉及文件：

- `Templates/Export/invoice_template.html`
- `Templates/Export/packing_list_template.html`
- `src/ExportDocManager.Infrastructure/Services/Reporting/ReportTemplateStarterFactory.cs`

展示规则：

| 字段 | 修改前 | 修改后 |
| --- | --- | --- |
| 订单号 | `PO: 2024AA001` | `2024AA001` |
| 款式号 | `Style: YLAW1320` | `YLAW1320` |

这不是删除订单号或款式号。Scriban 字段引用仍为 `item.PoNumber` / `item.StyleNo`，默认文件模板和管理员新建模板的起始内容保持一致。这样客户可直接使用纯数据，也可在自定义模板中自行添加 `订单号：`、`款号：`、`PO#` 等本地化标签。

多商品 PDF 基线因文本变短而更紧凑：出口发票由 4 页收敛为 3 页，装箱单由 5 页收敛为 3 页；方向、内容、金额/重量数据和跨业务域隔离不变。

## 3. 独立账号管理页面

独立入口位于“系统维护 → 账号与权限”，路由为 `/system/access-control`。页面复用现有账号与权限组件，不复制 API、状态或权限逻辑：

- “账号管理”：新增、编辑、启用/停用、岗位、权限模板、部门、公司范围、初始/重置密码、删除。
- “权限模板”：按业务模块配置仅查看、可操作、可管理，并由服务端补齐技术依赖。
- 仅 `Full` 管理员显示导航；普通账号、`Document`、`Sales` 版本不可见且直达路由会被管理员/版本门禁拒绝。
- 删除使用项目统一确认对话框，不再调用浏览器原生 `window.confirm`。

服务端保护保持并继续验证：不能停用当前管理员、不能取消自己的管理员身份、不能删除当前登录账号、不能删除最后一个启用管理员、已有发票或付款归属的账号只能停用。

## 4. 注册与首管理员方案

当前产品是企业内部业务系统，不采用自由注册：

1. 管理员完成容器部署后访问登录页，第一次使用用户名 `admin` 和自定密码登录。
2. API 仅在 PostgreSQL 用户表为空且用户名为 `admin` 时，用本次密码创建首个管理员；其他用户名不能触发初始化。
3. 管理员登录独立账号页面创建业务、单证、财务或其他管理员账号。
4. 人员离职或暂时停用时优先停用账号；无业务归属且满足安全规则时才允许删除。

选择该方案的原因：内部系统必须先确定公司、岗位、权限模板和数据归属；自由注册无法可靠决定这些信息，也会增加垃圾账号和越权风险。未来若发展外部客户门户，应新增一次性邀请令牌、过期时间和邀请范围，不与当前员工工作区共用公开注册入口。

## 5. 密码安全基线

- 普通账号初始或重置密码：服务端强制 8–128 个字符，前端同步提示和预校验。
- 容器首管理员：首次 `admin` 登录时服务端同样强制 8–128 个字符；该密码与 PostgreSQL 数据库连接密码相互独立。
- 哈希：PBKDF2-SHA256，16 字节随机盐、32 字节派生密钥、210,000 次迭代。
- 校验：使用 `CryptographicOperations.FixedTimeEquals`；损坏或非法哈希返回认证失败，不抛出格式异常导致 500。
- 密码哈希不进入 API DTO；审计拦截继续脱敏密码字段。

## 6. 存储与依赖边界

- 用户、角色、权限模板和账号状态只写当前 SQLite/PostgreSQL 运行数据库。
- 容器 PostgreSQL 数据位于运行根 `postgres/`，API 数据位于 `api-data/`，配置位于 `config/appsettings.json`；首管理员密码不新增 `.env` 字段，只保存数据库哈希。
- 不写 LocalStorage、SessionStorage、IndexedDB、AppData、ProgramData 或硬编码系统 C 盘路径。
- Web/Tauri 共用同一 React 页面和 API；仅桌面特有的更新器继续保持桌面专属，账号管理没有平台分叉。
- 继续使用现有 React、TanStack Query、Lucide 和 ASP.NET/EF Core 能力，不增加重量级身份框架或云账号服务。
