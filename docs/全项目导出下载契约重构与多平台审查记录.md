## 1. 审查结论

本轮已完成全项目导出、下载、上传导入和服务器路径写入契约的统一重构。当前代码只保留未投产阶段的唯一新契约，不保留旧 URL、旧 operationId、旧 DTO、可选 `destinationPath` 猜测运行形态或前端兼容适配层。

当前规则如下：

1. Web/容器端不显示、不接收、不返回服务器绝对路径。
2. 小文件或可快速生成文件直接返回标准浏览器附件。
3. 长耗时输出采用“创建任务 -> 查询状态 -> 下载任务结果”。
4. Tauri 桌面端使用原生文件对话框和独立 `save-to-path` 端点。
5. 所有服务器路径写入端点都在端点内部显式校验 desktop token，不能只依赖全局中间件。
6. 浏览器任务输出只进入运行数据根 `Exports/Browser/`，浏览器上传暂存只进入运行数据根 `Cache/BrowserUploads/`。
7. 数据库、缓存、日志、上传暂存、任务快照和浏览器导出均不新增系统 C 盘、AppData、ProgramData 或系统临时目录落点。

本专项代码契约完成度为 `100%`。真实 Docker/PostgreSQL、反向代理、大文件并发和 macOS/Linux 实机属于部署验收，不计入本次代码完成度，也不得据此标记生产就绪。

## 2. 统一交互模型

### 2.1 浏览器直接下载

适用于内存可控、生成时间短或已经形成单一文件的功能。API 返回正确 MIME、`Content-Disposition: attachment` 和文件名，前端使用统一 Blob 下载辅助函数交给浏览器默认下载目录。

当前包括：

- 查询结果 Excel：`POST /api/query/invoices/download`
- 审计日志 Excel：`POST /api/audit-logs/download`
- 报表模板包：`POST /api/reports/templates/package/download`
- 支持包：`POST /api/support-package/download`
- 发票转移包：`POST /api/invoices/{id}/transfer-package/download`
- 单一窗口 COO/ACD 提交包下载
- 单一窗口回执包下载

支持包下载完成后立即尽力删除服务器生成的 ZIP，不把诊断包长期留在服务器。

### 2.2 浏览器后台任务下载

适用于 PDF 渲染、批量 ZIP、单据包、Excel 转换和 PDF 合并等可能耗时或需要进度反馈的操作：

```text
POST 业务 download/upload 端点
  -> 202 + jobId
  -> GET /api/jobs/{jobId}
  -> Succeeded
  -> GET /api/jobs/{jobId}/download
```

任务输出路径固定为：

```text
运行数据根/Exports/Browser/{kind}/{guid}/{fileName}
```

普通 Web 用户的任务列表和详情只返回输出文件名，不返回绝对路径、重试请求 JSON 或服务器目录结构。可信 Tauri 请求才允许读取完整任务路径。任务下载按 `RequestedBy` 和管理员范围授权；普通用户读取其他用户任务或结果时返回 `404`，避免泄露任务是否存在。

删除任务或清理已结束任务时，会尽力删除对应的受控浏览器输出目录。

### 2.3 Tauri 桌面保存

桌面端继续使用原生文件/文件夹对话框，用户明确选择目标后调用独立 `save-to-path` 端点。路径字段只在 `isDesktopBridgeAvailable()` 成立时渲染。

即使用户已经以管理员 Bearer 登录，缺少有效 desktop token 的路径端点仍返回 `403`。这一规则覆盖查询、审计日志、发票/付款报表、批量 ZIP、单据包、Excel 工具、PDF 合并、转移包、模板包、支持包和单一窗口包等相关入口。

## 4. 浏览器上传与暂存策略

浏览器不能把本机文件路径传给服务器。需要输入文件的功能统一使用标准上传：

| 功能 | 浏览器端点 | 暂存目录 |
| --- | --- | --- |
| Excel 转托单 | `/api/tools/excel/booking-sheet/convert/upload` | `Cache/BrowserUploads/Excel/{guid}` |
| PDF 合并 | `/api/tools/pdf/merge/upload` | `Cache/BrowserUploads/PdfMerge/{guid}` |
| 发票转移包预览/导入 | `/api/invoices/transfer-package/upload/preview`、`upload/import` | `Cache/BrowserUploads/InvoiceTransfer/{guid}` |
| 单一窗口提交包/回执包导入 | `/api/single-window/packages/upload`、`receipts/upload` | `Cache/BrowserUploads/SingleWindow/{guid}` |
| 报表模板包导入 | `/api/reports/templates/package/upload` | 既有运行数据根受控上传边界 |

上传暂存目录在任务完成、失败或取消后尽力清理。PDF 合并复制上传文件期间若发生异常或请求取消，也会立即清理本次上传目录。上传任务不保留无法复现源文件的“重试”按钮；用户需要重新选择文件发起新任务。

PDF 合并当前限制为 2 至 50 个 `.pdf` 文件，总大小不超过 100 MB。前端使用多文件选择器，服务端仍独立执行数量、扩展名和总大小校验。

## 5. 审计日志危险操作语义

“删除筛选结果”和“按保留期清理”不是重复功能：

- 删除筛选结果：只删除已经提交并执行查询的筛选条件所命中的记录；无筛选时服务端拒绝执行。它被放在折叠的高级操作中，适合删除测试账号、特定业务记录或指定时间段数据。
- 按保留期清理：忽略当前筛选条件，保留最近指定天数，删除更早的历史日志，适合日常维护。

两者都使用显示范围、数量和不可恢复说明的确认弹窗，不再要求输入 `DELETE`。API 使用当前 `confirmed: boolean` 契约，不保留 `confirmationText` 魔术字符串。

## 6. 未改动但已复核的功能

- CRM 客户 Excel 和供应商 Excel 已经是标准 Blob 下载，不接收服务器路径，无需改造成后台任务。
- WebDAV `/api/backup/cloud/download` 的语义是把云端备份下载到当前服务器运行数据根备份目录，不是给浏览器下载文件，因此保留现有服务器端命令语义。
- 数据库仍使用当前运行数据根 SQLite 或显式 PostgreSQL；本轮没有修改数据库 schema、迁移、连接配置或业务数据兼容逻辑。

## 7. 解耦与扩展边界

- API 路由只负责认证、运行形态门禁、请求校验和任务编排。
- 文件生成继续由 Application/Infrastructure 服务承担，不把 PDF、Excel、压缩包实现堆回端点。
- 浏览器任务路径、上传暂存、MIME、任务授权和文件清理由共享 helper 统一处理。
- OpenAPI 是唯一客户端契约来源，修改端点后重新生成 TypeScript 客户端，不维护手写旧方法适配器。
- 前端用统一 `downloadBlob` 和 `downloadJobResult` 处理直接下载与任务下载，各业务页面只负责用户意图和状态提示。
- 本轮未新增 npm、NuGet、Cargo 或 Python 依赖，继续使用现有免费开源技术栈。
