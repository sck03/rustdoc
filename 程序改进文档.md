## 1. 当前结论

项目主链路已经从旧 `WinForms 宿主` 收敛为 `Tauri 桌面壳 + React/Web 页面 + ASP.NET Core API sidecar + Application/Infrastructure 业务能力`。当前改进重点不再是补基础分层，而是继续补齐旧桌面功能在 Tauri/Web/API 的生产闭环、压缩大节点、删除未投产旧胶水、统一基础设施真源，并用测试守住关键业务路径。

项目尚未部署生产环境，因此本阶段遵循：

- 直接替换旧实现。
- 删除无调用方兼容壳。
- 依赖缺失在构造期暴露。
- 默认路径和默认装配用显式无参入口表达，不再用 `null` 作为默认哨兵。
- 读侧走读仓，写侧服务不保留整批查询 fallback。
- 文件、ZIP、偏好、HTML 临时文件和导出链路复用共享 helper。
- 桌面端默认运行数据、数据库、授权文件、模型、浏览器和稳定资源跟随程序运行目录或显式 `--data-root`，不静默落到 AppData/ProgramData 或系统 C 盘默认用户目录。

## 2. 已稳定的改造事实

### 显式依赖

- 核心业务服务、API endpoint、设置页、报表设计器、发票/商品页面、单一窗口操作中心、协同页和编辑器已固定通过 DI/API client/桌面命令注入真实依赖。
- 生产代码不再把无参构造、可省略服务参数、空读仓、空服务或 `?? new Xxx()` 当作第二装配路径。
- `ReportService / ReportDataAssembler` 固定要求真实 `IDbContextFactory<AppDbContext>`，模板目录测试也通过测试侧 SQLite 内存工厂表达依赖，不再让生产报表服务接受空数据库工厂。
- 数据库连接配置、文件对话框最近目录、本机偏好、列布局和分栏布局都要求显式服务。
- 装柜 ViewModel、打印预览窗、HS 编码选择窗、仪表盘和分页搜索基类不再保留服务级 `?.` 弱运行分支；缺少服务直接作为装配错误暴露。

### 读写边界

- 发票、付款、主数据、查询页和审计页的高频读取走读仓。
- 写服务只保留保存、删除、复制、冲突处理、编号和单据级读取。
- 查询和审计导出已走分页读取、分段写表和固定列宽。

### 文件与导出基础设施

- 原子文本/文件写入、短暂文件占用重试、ZIP 创建/解压、普通/可取消文件复制已由共享 helper 承接。
- 批量导出命名走 `BatchExportPathHelper`。
- 批量导出的 PDF 合并后处理已下沉到 `PdfMergeService`，批量导出主流程不再直接依赖 PdfSharp 页合并细节。
- WebView2 预览和 PDF 生成走共享临时 HTML 文件导航，并按限龄/限量清理。
- 单证章/报关章进入模板前统一归一为透明 PNG data URI。

### 单一窗口

- `COO / ACD` 一期本地闭环已落地：编辑、字段映射、XML、交接包、回执导入、状态回写和操作中心。
- 字段映射、XML 校验、文档持久化、跟踪服务、操作中心、交接包、手工客户端桥接和导出审查已按职责拆分。
- 编辑器工具动作、顶部摘要、提示折叠、审查导航、字段锁定、源资料差异和底层布局 helper 已共享化。
- 导出预检修复已在服务层直接基于当前草稿和默认草稿处理，不再实例化编辑器 ViewModel。
- 导出预检能力已拆成主入口、布局和状态/行为模块；表格列、动作按钮和预检选择逻辑不再堆在同一个页面文件中。

### 设置与工具能力

- 业务设置和本机偏好已拆成独立文件。
- 本机偏好写入按文件路径串行合并，降低多实例互相覆盖风险。
- 设置页和设置操作模型已按页签、布局构件、状态、持久化、连接测试和日志清理分段。
- OCR 已固定为 `Microsoft.ML.OnnxRuntime + PP-OCRv6 Small ONNX CPU x64`，运行时只检查 `OcrModels/PaddleOCR/V6` 随包模型目录；缺模型会直接提示缺失文件。识别字典从 `rec/inference.yml` 完整解析，并在初始化时校验模型类别数与字典对齐。
- PostgreSQL 共享库配置已改成结构化字段输入：`服务器 / 端口 / 数据库名 / 账号 / 密码 / 附加参数`；应用内部统一生成连接串，数据库密码按本地加密方式保存。
- PostgreSQL 团队库维护已改为物理维护口径：优先从运行目录 `Tools/PostgreSQL/bin/` 或 `EXPORTDOCMANAGER_POSTGRES_BIN` 查找 `pg_dump / pg_restore / psql`，手动和自动备份默认写运行数据根 `App_Data/Backups/PostgreSQL/`，自动调度状态写 `App_Data/Backups/PostgreSQL/schedule-state.json`；设置页支持每天/每周、执行时间和保留最近 N 份；还原计划默认写 `App_Data/Backups/PostgreSQL/RestorePlans/`，生成 `pg_restore` 脚本和 `post_restore_ownership.sql`，覆盖 `REASSIGN OWNED`、`ALTER OWNER`、`GRANT` 和默认权限修复流程；旧应用级共享库快照端点已删除。
- 崩溃诊断支持包为离线导出，默认只收集程序版本、构建/运行路径、数据库模式摘要、脱敏配置、后台任务快照和最近日志；Tauri panic/startup 错误落盘到 `logs/tauri-errors.log`，前端全局错误、未处理 Promise 和 React ErrorBoundary 落盘到 `logs/frontend-errors.log`，支持包优先收集 API sidecar、前端和 Tauri 关键日志；数据库备份或样张文件必须由管理员明确勾选并输入 `INCLUDE OPTIONAL FILES` 后才打包。
- SQLite / PostgreSQL 切换时，系统设置页会在当前标签页内立即重排，不再需要切走再切回才能刷新显示。
- 数据库事务已统一到 `AppDbContextExecution.ExecuteInTransactionAsync(...)`。发票保存/删除/复制、单据包导入、装柜方案保存、审计日志批删和单一窗口保存/跟踪写入都不再直接手写 `BeginTransactionAsync()`，从而兼容 PostgreSQL provider 启用连接重试后的执行策略要求。
- 数据库默认配置入口已继续收紧：默认读取走 `DbHelper.LoadDatabaseSettings()`；指定路径、指定连接配置和数据库模式判断必须传真实对象，不再把 `null` 解释为默认 SQLite。
- F5 调试态与发布态的性能策略已区分：调试器附加时默认不写 VS Debug 输出，也不启动主工作区隐藏页后台预热，避免 Visual Studio 把日志输出、句柄创建和布局递归成本放大成“界面卡住”；发布/直接运行仍保留原有文件日志和后台预热。
