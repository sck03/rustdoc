# 多平台与多架构支持矩阵

> 最近更新: 2026-07-17
>
> 适用主线: `Tauri v2 + React/Vite + ASP.NET Core API sidecar + Domain/Application/Infrastructure`

## 1. 当前结论

项目代码主线已按 Windows、Linux、macOS 与 x64、ARM64 组合维护，不再把 Windows x64 当成唯一编译事实源。

- React/Web 页面为三平台共享源码，响应式断点覆盖桌面、中等宽度和窄窗口。
- Tauri 主窗口使用平台无关 `Path/PathBuf` 与 `WebviewWindowBuilder::data_directory(...)`，WebView 配置、缓存和浏览器状态固定进入运行数据根 `WebView/`。
- Tauri `runtime-paths.json` 使用同目录原子替换和 `.bak` 恢复，严格校验 schema；损坏配置不会静默切换到默认空数据库，未来 schema 也不会错误回退旧配置。
- .NET 普通构建根据当前宿主操作系统和进程架构选择 `win/linux/osx × x64/arm64` RID，避免把全部 native runtime 重复复制到每个 `bin`；交叉验证和发布可显式传入 RID 覆盖。
- `prepare-tauri-bundle.mjs` 支持 `win-x64`、`win-arm64`、`linux-x64`、`linux-arm64`、`osx-x64`、`osx-arm64` sidecar RID。
- GitHub Actions 已建立 Windows、Linux、macOS 原生 Tauri 检查、各平台 x64/ARM64 Infrastructure 编译契约，以及运行数据根/依赖诊断/系统目录禁用专项测试。
- 容器数据库默认使用官方 `postgres:18-bookworm` Linux 多架构镜像，已确认覆盖 amd64 与 arm64/v8；Windows/macOS Docker Desktop 通过 Linux VM 运行对应架构镜像。数据库初始化使用 PostgreSQL 内置 `PG_UNICODE_FAST` Unicode locale，不依赖容器 glibc/musl 默认排序。

## 3. 跨平台边界

- 业务层不得引用 WinForms、`Microsoft.Web.WebView2`、`System.Drawing.Common`、Registry、WMI 或平台文件对话框。
- 文件与目录使用 `Path`、`PathBuf`、`path.join` 和 Tauri desktop command，不拼接反斜杠作为业务逻辑。
- 稳定资源位于程序根；数据库、缓存、安全文件、WebView 数据和业务可写数据位于运行数据根；用户导入导出使用显式选择路径。
- Windows `WebView2Loader.dll` 只在 Windows bundle 中要求；Linux/macOS 不复制该文件。
- OCR、浏览器渲染器、PostgreSQL 工具和安装器属于平台适配资产，按 RID/平台分别打包，不进入 Domain/Application。
- Tauri/Web 客户端不直接加载 PostgreSQL 容器 libc：浏览器/Tauri 访问 ASP.NET Core API，API 通过 Npgsql 网络协议访问 PostgreSQL。数据库镜像 libc 只影响服务器端扩展和 libc locale；当前默认改用 Bookworm/glibc，并以 PostgreSQL builtin locale 固定文本行为。

## 4. 当前验证入口

- Web: `npm --prefix apps/export-doc-web run build`
- Tauri 当前平台: `cargo check --manifest-path apps/export-doc-tauri/src-tauri/Cargo.toml`
- Tauri 路径契约: `cargo test --manifest-path apps/export-doc-tauri/src-tauri/Cargo.toml`
- .NET 路径契约: `dotnet test tests/ExportDocManager.Infrastructure.Tests/ExportDocManager.Infrastructure.Tests.csproj --filter "FullyQualifiedName~RuntimeAppPathProviderTests|FullyQualifiedName~InfrastructurePathPolicyTests|FullyQualifiedName~RuntimeDependencyDiagnosticsServiceTests"`
- 当前宿主 .NET: `dotnet build src/ExportDocManager.Api/ExportDocManager.Api.csproj`
- 显式架构: `dotnet build src/ExportDocManager.Infrastructure/ExportDocManager.Infrastructure.csproj -r <rid>`
- 自动矩阵: `.github/workflows/cross-platform-validation.yml`

## 5. 后续验收

1. 在 Windows ARM64、Linux ARM64 和 macOS ARM64 真实设备上运行 Tauri 主程序与 sidecar。
2. 为 Linux/macOS/ARM64 准备对应 Chrome Headless Shell 或明确采用系统 Chromium 的受控策略。
3. 评估 PP-OCRv6 在 Linux/macOS 与 ARM64 的 ONNX/OpenCV native runtime，未验证前保持明确“不支持”，不静默降级。
4. 分平台完成 Tauri updater endpoint、签名、公证、安装升级、失败回滚和卸载测试。
5. PostgreSQL 团队版按平台提供或显式配置 PostgreSQL 18 的 `pg_dump/pg_restore/psql`，客户端工具主版本不得低于服务器主版本，不假设 Windows `.exe` 文件名。
